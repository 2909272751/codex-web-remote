# Codex Web Remote v1.4.1

## 修复

- 修复新建的空任务在发送第一条消息前无法点击打开的问题。
- 对 Codex “尚未实体化”的新任务改用轻量读取；第一条消息发送后自动恢复完整历史。
- 新增创建后立即打开空任务的回归测试。

---

## v1.4.0

本版本补齐 Web 项目管理和远程主机目录选择。

## 新功能与修复

- 修复 Web 只能沿用旧任务目录、无法添加新项目的问题。
- 侧栏新增项目列表，可按项目筛选本机任务。
- 新增主机目录浏览器，手机、iPad 和远程电脑均可浏览 Windows 磁盘并选择项目文件夹。
- 添加项目不要求先接管 Codex；选择项目后，新任务会明确创建在该目录中。
- 项目名称和路径持久保存，重启及覆盖更新后仍然保留。
- 从列表移除项目只移除 Web 入口，不删除源码文件或 Codex 历史记录。
- 服务端校验项目目录必须存在，避免任务误建到程序安装目录。
- 优化桌面、平板和手机上的项目列表与目录选择弹窗布局。

## v1.3.0

本版本加入 GitHub 自动更新和一键安全升级。

## 新功能

- 托盘控制中心启动时及每 6 小时检查 GitHub 最新 Release。
- 远程 Web 检测到新版后显示更新横幅和版本说明链接。
- 托盘与远程 Web 均可发起一键更新。
- 自动下载 Setup 与 SHA256 文件，校验一致后才进入安装。
- 使用安装目录外的临时更新器，安全停止网关、覆盖当前安装目录并自动重启。
- 有任务运行、正在启动或仍有排队消息时，Web 会拒绝更新，避免任务中断。
- 更新保留密码、端口、开机自启、88frp 地址、会话和浏览器数据。

## 说明

Windows 无法在进程运行期间真正热替换 EXE，因此这里采用“下载与校验期间不停机，安装阶段短暂重启”的准热更新方式。

## v1.2.0

本版本加入多 Web 设备共享控制。

## 新功能

- 手机、iPad 和电脑可以使用各自的 24 小时登录会话同时控制 Web Codex。
- 不再出现“另一个浏览器正在控制”导致的长期占用。
- 两个设备同时向同一任务发送消息时，服务端会自动串行：一条立即运行，另一条进入队列。
- 所有设备共享任务状态、思考进度、审批、排队消息和文件上传结果。
- 终端模式继续保持单会话输入，避免多个键盘同时操作同一终端。
- 更新 PWA 缓存版本，升级后手机会获取新的共享控制界面。

## 升级

直接运行 v1.2.0 安装包覆盖安装即可，密码、端口、自启和 88frp 地址都会保留。

## v1.1.1

这是一个接管启动兼容性修复版本。

## 修复

- 修复新电脑没有 `node_repl` 和 `openaiDeveloperDocs` MCP 配置时，接管会报 `Codex app-server stopped (code=1, signal=none)` 的问题。
- app-server 意外退出时，网页错误和托盘日志现在会显示退出前的真实错误，便于排查不同电脑上的环境问题。
- 新增干净 Codex 配置回归测试，确保安装包不依赖开发电脑独有的配置。

## 安装

已安装 v1.1.0 的电脑直接运行 v1.1.1 安装包覆盖升级即可，密码、端口和 88frp 地址会保留。

## v1.1.0

本版本将安装和日常管理改成完全图形化流程，普通用户不再需要 PowerShell、Node.js 或命令行。

## 新功能

- 中文 `Setup.exe`，双击后按安装向导完成安装
- 原生 Windows 首次设置界面
- 托盘控制中心：启动、停止、重启、打开网页和查看日志
- 密码使用 Windows DPAPI 加密保存
- 图形化修改密码、端口、HTTPS Cookie 和开机启动
- 88frp 配置提示、公网地址保存和二维码
- PWA 支持，可在手机和 iPad 添加到主屏幕
- 配置与程序文件分离，覆盖升级不会丢失设置
- 标准卸载流程，卸载不会删除 Codex 原始任务

## 安装

1. 安装并登录 Codex Windows App，确认能正常打开任务。
2. 下载 `CodexWebRemote-Setup-1.1.1-win-x64.exe`。
3. 双击安装，安装完成后会自动打开首次设置界面。
4. 设置 Web 密码，点击“完成设置并启动”。
5. 本机打开 `http://127.0.0.1:18888`；远程访问时，将 88frp 指向 `127.0.0.1:18888`。

免安装用户可以下载 `CodexWebRemote-Portable-1.1.1-win-x64.zip`，解压后双击 `CodexWebRemote.exe`。
